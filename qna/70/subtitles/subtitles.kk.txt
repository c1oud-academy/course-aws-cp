Proxy Server дегеніміз не және ол қалай жұмыс істейді? Жауап берейін.
Proxy Server - бұлтты технологияларға тіркелмеген жалпы IT термині.
Қалай болғанда да, жалпы түсінік болуы үшін мен сізге қарапайым сөзбен жауап беремін.
Браузер арқылы белгілі бір веб-сайтқа қосылған кезде, бұл веб-сайт интернеттегі сіздің IP-мекенжайыңызды көреді.
Сондай-ақ, қосымша ақпаратты, мысалы, сіз тұратын ел және т.б. ақпаратты ала алады.
Proxy Server туралы айтатын болсақ, бұл Интернет пайдаланушылары мен сайттар арасындағы аралық байланыс түрі.
Осылайша, сіз Интернет пайдаланушысы ретінде алдымен IP мекенжайыңызбен Proxy Server-ге қосыласыз.
Әрі қарай, сұрауыңыз соңғы веб-сайтқа сол түрде таратылады, бірақ байланыс прокси серверден келеді.
Осылайша, веб-сайт сіздің IP мекенжайыңызды емес, Proxy Server-дің IP мекенжайын көреді.
Сонымен қатар, қосымша ақпаратты: осы Proxy Server орналасқан елді көреді.
Proxy Server әдетте қашан пайдаланылады? Сіз А елінде екеніңізді елестетіп көрейік, және сізге қандай да бір сайтқа қосылу керек, бірақ сізде тікелей кіру мүмкіндігі жоқ, оны Интернет провайдерлері бұғаттады.
Бұл сайтқа кіру үшін Proxy Server арқылы сұрау жіберу керек.
Осылайша Proxy Server-ге қосыласыз.

Өзгертуге болмайтын hard limits және баптауға болатын soft limits бар.
Осы және басқа опцияларды сәйкес қойындыда көресіз.
Amazon EC2 сервисін AWS Management Console-інде ашсаңыз, сол жақ шарлау мәзірінде Limits қойындысы болады.
Біз мұны зертханалық жұмысымызда жасаймыз, толығырақ сол жерде талқылаймыз.
Осы Limits қойындысына жеткенде Amazon EC2 сервисі аясындағы барлық шектеулерді көресіз.
Қажет болса немесе сіз қазір осындай ресурс мәнін пайдаланып жатсаңыз немесе Limit-ке жақын кез келген нәрсе болса, Support қызметі арқылы осы soft limits-тарды әрқашан көбейте аласыз.

Ephemeral volume дегеніміз не? Жауап. Ephemeral volume - бұл уақытша қатқыл диск.
Ол сізге Amazon EC2 сервисінің Instance Store мүмкіндігін пайдаланған кезде қолжетімді болады.
Бұл ephemeral volume бір серверде орналасқан физикалық қатқыл диск болып табылады және осы серверде Amazon EC2 сервисінің бөлігі ретінде сіз жедел жадының бөлігін және процессор ядроларының санын аласыз.
Бұл - операциялық жүйелер Amazon EC2 сервисінің аясында орнатылған бірінші сақтау опциясы.
Содан кейін, біраз уақыттан кейін, айтарлықтай жақсартуды қамтамасыз ететін жаңа Amazon EBS сервисі шығарылды, бұл best practice және Root Volumes үшін AWS ұсынған опция.
Root Volumes - операционды жүйені орнататын қатқыл диск.
Сіз әрқашан біздің дәрістерімізде олардың қалай ерекшеленетінін көре аласыз, сонымен қатар қажетті парақшаға өтіп, көбірек білу үшін іздеу жүйесіне бір сұрау арқылы қосымша ақпаратты ала аласыз.

Идеясы, сіз икемді IT ресурстарымен қамтамасыз етілесіз.

Бұл деректерді CloudWatch сервисіне жіберуді өшірсек, Amazon EC2 Instance көрсеткіштеріне не болады? Олар сақталады ма, әлде жоғалады ма?

EC2 сервисі аясында метрикаларды CloudWatch сервисіне тасымалдамасаңыз, бұл деректер ол жерде болмайды, сіз бұл деректермен жұмыс істей алмайсыз.
Бұл жағдайда сізде екі нұсқа бар, немесе метрика құрмаңыз және бақылауды орнатпаңыз, бұл өте жаман кеңес.
Мұны тек сынақ немесе Dev ортасына жататын жұмыс жүктемелері үшін жасауға болады, бірақ Production ортасында ол әрқашан бақылаумен бапталынуы керек, сіз инстанстардың күйін қадағалап тұруыңыз керек.
Басқа нұсқа, егер қандай да бір себептермен CloudWatch сервисін пайдалану сізге сәйкес келмесе, онда сіз өзіңізге сай мониторингті орната аласыз.
Бұл мониторинг деректерді, ең алдымен, осы инстанстың ішінде сақтайды және сіз бағдарламаны осы көрсеткіштер тартылып, бағдарламалық құралдағы басқа бағдарламада көрсетілетіндей етіп конфигурациялай аласыз.

CloudWatch сервисі көрсеткіштерді сақтауға және инстанстарыңыз бен басқа сервистерді болашақта бақылауға арналған best practice және ұсынылған тәсіл болып табылады.

Бұл ақпараттың барлығы метрика түрінде беріледі және осы көрсеткіштердегі өзгерістерді графиктер түрінде көруге болады.
Логтар туралы айтатын болсақ, олар қандай да бір процестерді өңдеу үшін белгілі бір пәрмендерді іске қосқаннан кейін шығарылатын ақпарат болып табылады.

Қолданба логтары деген кезде, яғни қолданбаңыз инстанстың ішінде жұмыс істейді және кейбір журналдарды жасайды.
