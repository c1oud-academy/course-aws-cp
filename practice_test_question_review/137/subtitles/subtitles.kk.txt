Қайырлы күн, құрметті студенттер! Сіздердің барлығыңызды келесі Practice Test сұрақтарын талдау сессиясында көргеніме қуаныштымын. Practice Test сұрақтары нақты AWS емтихан сұрақтарына жақын сұрақтар болып табылады. Осылайша, біздің платформадағы Practice Test-термен жаттығып, сондай-ақ барлық Practice Test сұрақтарын талдау арқылы сіз нақты AWS емтиханын сәтті тапсыру мүмкіндігіңізді арттырасыз. Талдау үшін біз ең қиын сұрақтарды таңдаймыз, сондықтан осы сұрақтарға қалай жауап беру керектігін біліп алсаңыз, басқа сұрақтарға жауап беру қиындық тудырмайды. Сондай-ақ, сіз сұрақтың мазмұнына ғана емес, сонымен қатар дұрыс жауапты табу үшін қандай әрекеттер жасайтынымызға да назар аударуыңыз керек екенін атап өткім келеді. Сонымен, бастайық.
Сұрақ: “Nitro гипервизор деп аталатын EC2 инстансының гипервизорымен өнімділік жағынан проблемалар туындады. Бұл проблеманы түзету керек. Сізден бұл мәселені шешу сұралады. Сіз не істейсіз?” Осы жауап нұсқаларында бір дұрыс жауапты таңдауыңыз керек. Жақсы, бұл сұраққа жауап беруге тырысайық. А нұсқасына қарайық: “Update the Nitro Hypervisor to a new version”. Естеріңізде болса, гипервизор – бұл физикалық қуатты серверді қуаты аз виртуалды серверлерге бөлуге мүмкіндік беретін технология. Осылайша, гипервизор AWS жаһандық инфрақұрылымының бөлігі болып табылады. Бұл AWS өзі басқаратын технология, біз бұл гипервизорға жете алмаймыз, яғни біз соңғы нәтижені көріп отырмыз. Осылайша, А нұсқасы - дұрыс емес.
Келесі нұсқаны қарастырайық: “Migrate to a different version of the hypervisor”. Бұл А нұсқасының басқаша жазылған аналогы секілді. Жауабы тура солай. Гипервизор – біз оны өзгерте немесе жаңарта алмаймыз, сондықтан біз В нұсқасын алып тастаймыз.
D нұсқасын қарастырайық: "Call up AWS Support and complain about the issue". Бұл нұсқа логикалық тұрғыдан дұрыс болуы мүмкін. Оны сары түспен белгілеп, C нұсқасын көрейік: “Nothing, this is AWS responsibility as per the Shared Responsibility Model”. Жауап нұсқасында Shared Responsibility Model туралы айтылған. Осылайша, жалпы алғанда, сұрақ Shared Responsibility Model терминін дұрыс түсінетінімізді тексереді. Shared Responsibility Model-і аясында, егер AWS бір опция үшін жауапты болса, біз араласпаймыз. AWS барлығын жауапкершілікпен өзі шешуі керек. Егер Shared Responsibility Model-і аясында, сол опция үшін біз жауапты болсақ, онда AWS бізге араласпайды және біз IT-архитектурамыздың компонентін өзіміз дұрыс баптауымыз керек. Бұл жағдайда гипервизор бізге тиесілі емес. Осылайша, C нұсқасы бұл сұрақтың дұрыс жауабы болады.
Жалпы, AWS бұлтында қандай да бір проблемалар туындаса, біріншіден, ең жылдам шешімі – Интернетке шолу жасау, мүмкін басқалар осындай проблемаға тап болды және сіз жылдам жауап таба аласыз. Егер сіз бір-екі сағат ішінде жауап таба алмасаңыз, онда дұрыс және ұтымды шешім – бір мезгілде AWS Support-қа өзіңіздің проблемаңыз туралы жазып, осы мәселенің шешімін іздеуді жалғастыру керек. Кейбір жағдайларда AWS тез жауап береді және бұл мәселені шешуге көмектеседі. Бұл мәселе тіпті жасырын және AWS ішіндегі қате болып табылатыны өте сирек кездеседі, AWS Support танып, бұл қате шешілгенге дейін бізді бақылауда ұстайды. Бірақ көбінесе біз жауапты Интернеттен таба аламыз, егер біз шешімін мүлдем таба алмасақ, біз AWS Support-тан шешімін қайдан табуға болатыны туралы жолдама аламыз, осылайша, мәселе шешіледі.
Осымен біз Practice Test сұрағын талдау сессиясын аяқтаймыз. Сіз жаңа ақпарат білдіңіз деп үміттенемін. Келесі белсенділіктерімізде көріскенше!
