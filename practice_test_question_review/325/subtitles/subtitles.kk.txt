Қайырлы күн, құрметті студенттер! 
Сіздердің барлығыңызды келесі Practice Test сұрағын талдау сессиясында көргеніме қуаныштымын.
Practice Test сұрақтары нақты AWS емтихан сұрақтарына жақын сұрақтар болып табылады.
Осылайша, біздің платформадағы Practice Test-тер бойынша жаттығып, сондай-ақ барлық Practice Test сұрақтарын талдау арқылы сіз нақты AWS емтиханын сәтті тапсыру мүмкіндігіңізді арттырасыз.
Біз талдау үшін ең қиын сұрақтарды таңдаймыз, сондықтан сіз осы сұрақтарға қалай жауап беру керектігін білгеннен кейін, қалғандарына жауап беру қиын болмайды.
Сұрақтың мазмұнына ғана емес, сонымен қатар дұрыс жауап табу үшін қандай әрекеттер жасайтынымызға назар аудару керек екенін атап өткім келеді.
Сонымен, бастайық.
Сұрақ: «Сіз AWS жүйесінде WordPress орнатып жатырсыз.
Сізге веб-сервер жалпыға қолжетімді болуы керек, бірақ дерекқор жасырын болуы және интернеттен қолжетімді болмауы керек.
Осыған жету үшін сіз қандай әрекеттер орындауыңыз керек?»
Бұл сұрақта сіз бір дұрыс нұсқаны таңдауыңыз керек.
Бұл сұрақ қорқынышты болып көрінуі мүмкін.
Неге?
Өйткені жауап нұсқалары ұзын.
Сұрақтың өзі де ұзын болуы мүмкін жағдайлар болады.
Мен алдында бірнеше рет айтқанымдай, мұндай сұрақтардан қорықпау керек.
Керісінше, мұндай сұрақ келгенде қуану керек, өйткені көптеген егжей-тегжейлер жеткізіледі және сіз кейбір жауап нұсқаларын анық алып тастай аласыз.
Сұрақтың негізгі түйіні мынада: веб-сервер жалпыға қолжетімді болуы керек, ал дерекқор Интернеттен қол жетімсіз болуы керек.
Бұл веб-сервер public subnet-те, ал дерекқор private subnet-те болуы керек дегенді білдіреді.
Қандай жауап нұсқалары бар екенін көрейік.
A нұсқасы - In a VPC create a private subnet and place both, web server and database, in it.
No special security group configuration is needed.
Бұл нұсқа сәйкес келмейтіні анық.
Біз тіпті осы нұсқаның жартысынан азын оқып, оны түсінеміз.
Неге?
Өйткені веб-сервер мен дерекқорды private subnet-те орналастыру ұсынылады.
Бұл жағдайда веб-серверге Интернеттен кіру мүмкін болмайды, бұл нұсқа бізге сәйкес келмейді.
B нұсқасы - In a VPC create two separate public subnets, place a web server in one public subnet and the database in another public subnet.
Allow the web server to access the database using a correctly configured security group.
Бұл жағдайда біз екі бөлек public subnet құруымыз керек, біреуіне веб-серверді, екіншісіне дерекқорды орналастыруымыз керек және веб-сервер дерекқорға қол жеткізе алатындай қосымша security тобын баптауымыз керек дейді.
Бұл нұсқа да дұрыс емес, өйткені дерекқорды public subnet-те орналастырсақ, біз оны бүкіл Интернетке қол жетімді етеміз, бұл біздің сұрағымызға және жалпы алғанда, бұлтта архитектураны құрудағы best practices-ке қайшы келеді.
Сондықтан біз бұл В нұсқасын алып тастап, әрі қарай қарайық, С нұсқасы - In a VPC create public subnet and place both web server and database in it.
No special security group configuration is needed.
Бұл нұсқада жалғыз шындық - security тобын баптаудың қажеті жоқ, өйткені екі ресурс бір subnet-те орналасқан.
Бірақ екі ресурсты да public subnet-те орналастыру ұсынылатындықтан, бұл нұсқа да дұрыс емес.
Неге?
Өйткені дерекқор Интернеттен қолжетімді болады.
Біз мұндай жағдайға жол бере алмаймыз.
Ал соңғы жалғыз нұсқа қалды және ол дұрыс нұсқа.
Не істеу керек екенін көрейік.
D - In a VPC create a public subnet for the web server and a private subnet for the database.
Allow the web server to access the database using a correctly configured security group.
Бұл дұрыс жауап және ол Интернеттен қолжетімді болуы үшін веб-сервер үшін бір public subnet-ті жасау керек екенін айтады.
Сондай-ақ дерекқорды орналастыру үшін private subnet құру керек және оған Интернеттен қол жеткізу мүмкін болмау керек.
Веб-сервер дерекқорға қол жеткізу үшін security тобын дұрыс баптауымыз керек.
Бұл нұсқа дұрыс жауап нұсқасы болып табылады.
Көріп отырғаныңыздай, сұрақ неғұрлым үлкен болса, соғұрлым жауап беру оңайырақ.
Ең бастысы - барлық жауап нұсқаларын баяу қарастыру, айырмашылықты анықтау және сұрақты шешу кезінде сәйкес келмейтін нұсқаларды алып тастау керек.
