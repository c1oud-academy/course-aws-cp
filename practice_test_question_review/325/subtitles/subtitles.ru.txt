Таким образом, потренировавшись на Practice Test-ах из нашей платформы, а также просмотрев все разборы вопросов, вы максимально увеличите свои шансы для успешной сдачи реального AWS экзамена. Для разбора мы выбираем самые сложные вопросы, поэтому научившись отвечать на эти сложные вопросы, ответить на остальные вопросы для вас не составит труда. Я бы хотел также отметить, что вам необходимо обратить внимание не только на содержание самого вопроса, но и также на то, какие действия мы предпринимаем для того, чтобы найти правильный ответ. Итак, давайте начнем. Вопрос: "Вы настраиваете WordPress на AWS. Вам необходимо, чтобы веб-сервер был публично доступен, но база данных должна быть скрыта и недоступна из интернета. Какие пути вы можете предпринять для этого?". В этом вопросе необходимо выбрать один правильный вариант. Этот вопрос может выглядеть устрашающим. Почему? Потому что длинные варианты ответов. У вас могут быть вопросы, где и сам вопрос большой. Как я несколько раз уже вам до этого говорил, не стоит бояться таких вопросов. Наоборот, надо радоваться, когда приходит такой вопрос, потому что очень много деталей передается, и вы явно можете исключить некоторые варианты ответов. Ключевой момент в вопросе это то, что веб-сервер должен быть публично доступен, а база данных должна быть недоступна из интернета. Это говорит о том, что веб-сервер должен быть в public subnet-е, а база данных в private subnet-е. Давайте посмотрим, какие есть варианты ответов. Вариант A - In a VPC create a private subnet and place both, web server and database, in it. No special security group configuration is needed. Этот вариант явно не подходит. Мы можем даже меньше половины этого варианта прочитать и уже это понять. Почему? Потому что предлагается веб-сервер и базу данных поместить в private subnet. В этом случае веб-сервер будет недоступен из интернета, этот вариант нам не подходит. Вариант B - In a VPC create two separate public subnets, place a web server in one public subnet and the database in another public subnet. Allow the web server to access the database using a correctly configured security group. В этом случае говорится, что нам необходимо создать два отдельных public subnet, веб-сервер положить в одну, базу данных положить в другую и необходимо настроить дополнительно security группу, чтобы веб-сервер мог обращаться к базе данных. Этот вариант также не является правильным вариантом, потому как положив базу данных в public subnet мы делаем ее доступной для всего интернета, что противоречит нашему вопросу и в целом best practices при построении архитектуры в облаке. Поэтому этот вариант B исключаем, двигаемся дальше, вариант C - In a VPC create public subnet and place both web server and database in it. No special security group configuration is needed. В этом варианте верно только то, что security группу конфигурировать не нужно, так как оба ресурса находятся в одном subnet. Но так как оба ресурса предлагается положить в public subnet, то этот вариант также не является верным. Почему? Потому что база данных станет доступным из интернета. Такое допустить мы не можем. И последний вариант, это единственный, который остался и является правильным. Давайте посмотрим, что же нужно сделать. D - In a VPC create a public subnet for the web server and a private subnet for the database. Allow the web server to access the database using a correctly configured security group. Это является правильным вариантом ответа и в ней говорится о том, что нам необходимо создать один public subnet для веб-сервера, чтобы он был доступен из интернета. Также создать private subnet для того, чтобы положить туда базу данных и он не был доступен из интернета. Для того, чтобы в веб-сервер мог обращаться к базе данных, нам необходимо корректно настроить security группу. Этот вариант является правильным вариантом ответа. Как вы могли убедиться, чем больше вопрос, тем легче на него ответить. Главное не спеша рассмотреть все варианты ответа, выявить разницу и обращаясь к вопросу, уже исключать варианты, которые заведомо не подходят. На этом мы заканчиваем разбор вопроса Practice Test-а.
